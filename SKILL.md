# LY Memory Palace — 记忆宫殿

> 你现在拥有一个**跨会话的长期记忆系统**。记忆不只是存储，而是一个有生命的网络——会生长、会关联、会衰减、会在需要时自动苏醒。

## SOP：每轮对话你必须执行的流程

### 步骤 1：被动唤醒（每轮对话开头）

**每当用户发来消息时**，你必须先执行以下命令来检索记忆：

```
exec: python {baseDir}/palace.py recall "用户消息的核心内容"
```

- 如果有唤醒的记忆 → **自然地融入回答中**，不要刻意提及"我从记忆中想起"之类的话，就像人类自然想起一样
- 如果没有唤醒 → 正常回答

### 步骤 2：正常回答用户

根据用户问题回答，如果步骤1有相关记忆，参考记忆来给出更个性化的回答。

### 步骤 3：判断是否需要记忆（每轮对话结尾）

回答完成后，判断本轮对话是否包含**值得长期记住**的新信息：

| 信息类型 | 存入层级 | 示例 |
|---------|---------|------|
| 用户偏好、规则、知识、技术栈 | `semantic` | 用户主要用Python，喜欢暗色主题 |
| 重要事件、决策、里程碑 | `episodic` | 3月12日决定重构数据库层 |
| 操作习惯、工作流、常用命令 | `procedural` | 用户习惯先写测试再写代码 |
| 普通对话/已有记忆重复 | **不记** | — |

如果有值得记住的信息，执行：

```
exec: python {baseDir}/palace.py store --layer <层级> --keywords "<关键词1>,<关键词2>" --content "<要记住的内容>" --emotion <neutral|important>
```

### 关键词质量要求

- **精准**：选择最能区分这条记忆的词，避免过于宽泛（如"代码"、"工作"）
- **2-5个**：太少不利于检索，太多会有噪音
- **中英文均可**：系统支持中英文混合关键词
- **具名优先**：项目名、工具名、人名等专有名词优先

### 重要注意事项

1. **不要重复记忆**：如果 recall 返回了类似内容，就不需要再 store
2. **不要过度记忆**：只记录真正有价值的信息，不是每句话都要记
3. **emotion=important 谨慎使用**：只用于用户明确表示重要、或关键决策/偏好
4. **不要告诉用户你在执行 recall/store**：这是你的内部流程，对用户透明

## 其他可用命令

当用户请求管理记忆时，你可以使用：

```bash
# 列出所有记忆
exec: python {baseDir}/palace.py list

# 列出指定层
exec: python {baseDir}/palace.py list --layer semantic --detail

# 查看统计
exec: python {baseDir}/palace.py stats

# 删除指定记忆
exec: python {baseDir}/palace.py remove --id <id>

# 手动衰减清理
exec: python {baseDir}/palace.py decay

# 导出备份
exec: python {baseDir}/palace.py export --file backup.json

# 导入记忆
exec: python {baseDir}/palace.py import --file backup.json
```

## 记忆强度说明

唤醒的记忆会有强度指示：
- `●●●●●` 非常强（最近频繁使用/重要标记）
- `●●●○○` 中等（偶尔使用）
- `●○○○○` 微弱（即将遗忘）

微弱记忆仍然有价值——它被唤醒说明确实相关，使用后会自动强化。

## 数据存储

所有记忆持久化存储在 `{baseDir}/data/` 目录下：
- `memories.json` — 记忆数据
- `hippocampus_index.json` — 海马体关键词索引

数据跨会话保留，不会因对话结束而丢失。
